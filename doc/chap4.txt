  
  [1X4 Groebner Basis[0X
  
  This  chapter  contains  the  declarations  and  implementations  needed for
  Groebner  basis. Currently, we do not provide algorithms to actually compute
  Groebner  basis;  instead, the declarations and implementations are provided
  here  for [5XGAP[0m objects and the actual elements of Groebner basis are computed
  by the [5XGBNP[0m package.
  
  
  [1X4.1 Constructing a Groebner Basis[0X
  
  [1X4.1-1 InfoGroebnerBasis[0m
  
  [2X> InfoGroebnerBasis_______________________________________________[0Xinfo class
  
  is the info class for functions dealing with Groebner basis.
  
  [1X4.1-2 GroebnerBasis[0m
  
  [2X> GroebnerBasis( [0X[3XA, rels[0X[2X ) _________________________________________[0Xfunction
  
  Arguments: [3XI[0m -- an ideal, [3Xrels[0m -- a list of relations generating [3XI[0m.
  
  [6XReturns:[0X  an    object   [3XGB[0m   in   the   `IsGroebnerBasis'   category   with
            `IsCompleteGroebnerBasis' property set on true.
  
  Sets  also [3XGB[0m as a value of the attribute GroebnerBasisOfIdeal for [3XI[0m (so one
  has an access to it by calling GroebnerBasisOfIdeal([3XI[0m)).
  There  are  absolutely no computations and no checks for correctness in this
  function.  Giving a set of relations that does not form a Groebner basis may
  result  in incorrect answers or unexpected errors. This function is intended
  to  be used by packages providing access to external Groebner basis programs
  and should be invoked before further computations on Groebner basis or ideal
  I.
  
  
  [1X4.2 Categories and Properties of Groebner Basis[0X
  
  [1X4.2-1 IsGroebnerBasis[0m
  
  [2X> IsGroebnerBasis( [0X[3Xobject[0X[2X ) ________________________________________[0Xcategory
  
  Arguments: [3Xobject[0m -- any object in [5XGAP[0m.
  
  [6XReturns:[0X  true when [3Xobject[0m is a Groebner basis and false otherwise.
  
  The  function only returns true for Groebner bases that has been set as such
  using the [10XGroebner Basis[0m function, as illustrated in the following example.
  
  [4X---------------------------  Example  ----------------------------[0X
    [4Xgap> Q := Quiver( 3, [ [1,2,"a"], [2,3,"b"] ] );[0X
    [4X<quiver with 3 vertices and 2 arrows>[0X
    [4Xgap> PA := PathAlgebra( Rationals, Q );[0X
    [4X<algebra-with-one over Rationals, with 5 generators>[0X
    [4Xgap> rels := [ PA.a*PA.b ];[0X
    [4X[ (1)*a*b ][0X
    [4Xgap> gb := GBNPGroebnerBasis( rels, PA );[0X
    [4X[ (1)*a*b ][0X
    [4Xgap> I := Ideal( PA, gb );[0X
    [4X<two-sided ideal in <algebra-with-one over Rationals, with 5 generators>, [0X
    [4X  (1 generators)>[0X
    [4Xgap> grb := GroebnerBasis( I, gb );[0X
    [4X<complete two-sided Groebner basis containing 1 elements>[0X
    [4Xgap> alg := PA/I;[0X
    [4X<algebra-with-one over Rationals, with 5 generators>[0X
    [4Xgap> IsGroebnerBasis(gb);[0X
    [4Xfalse[0X
    [4Xgap> IsGroebnerBasis(grb);[0X
    [4Xtrue [0X
  [4X------------------------------------------------------------------[0X

  [1X4.2-5 IsCompleteGroebnerBasis[0m
  
  [2X> IsCompleteGroebnerBasis( [0X[3Xgb[0X[2X ) ____________________________________[0Xproperty
  
  Arguments: [3XGB[0m -- a Groebner basis.
  
  [6XReturns:[0X  true when [3XGB[0m is a complete Groebner basis.
  
  While philosophically something that isn't a complete Groebner basis isn't a
  Groebner  basis  at  all, this property can be used in conjuction with other
  properties  to see if the the Groebner basis contains enough information for
  computations.  An example of a system that creates incomplete Groebner bases
  is `Opal'.
  [13XNote:[0m The current package used for creating Groebner bases is [5XGBNP[0m, and this
  package does not create incomplete Groebner bases.
  
  
  [1X4.3 Attributes and Operations for Groebner Basis[0X
  
  [1X4.3-1 CompletelyReduce[0m
  
  [2X> CompletelyReduce( [0X[3XGB, a[0X[2X ) _______________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis, [3Xa[0m -- an element in a path algebra.
  
  [6XReturns:[0X  [3Xa[0m reduced by [3XGB[0m.
  
  If [3Xa[0m is already completely reduced, the original element [3Xa[0m is returned.
  
  [1X4.3-2 CompletelyReduceGroebnerBasis[0m
  
  [2X> CompletelyReduceGroebnerBasis( [0X[3XGB[0X[2X ) _____________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis.
  
  [6XReturns:[0X  the  completely  reduced  Groebner basis of the ideal generated by
            [3XGB[0m.
  
  The  operation modifies a Groebner basis [3XGB[0m such that each relation in [3XGB[0m is
  completely     reduced.     The     [10XIsCompletelyReducedGroebnerBasis[0m     and
  [10XIsTipReducedGroebnerBasis[0m  properties are set as a result of this operation.
  The  resulting  relations  will  be  placed in sorted order according to the
  ordering of [3XGB[0m.

  [6XReturns:[0X  the element [3Xa[0m tip reduced by the Groebner basis.
  
  If [3Xa[0m is already tip reduced, then the original [3Xa[0m is returned.
  
  [1X4.3-4 TipReduceGroebnerBasis[0m
  
  [2X> TipReduceGroebnerBasis( [0X[3XGB[0X[2X ) ____________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis.

  [1X4.3-5 Iterator[0m
  
  [2X> Iterator( [0X[3XGB[0X[2X ) __________________________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis.
  
  [6XReturns:[0X  an  iterator  (in  the  IsIterator  category,  see the [5XGAP[0m manual,
            chapter 28.7).
  
  Creates  an iterator that iterates over the relations making up the Groebner
  basis.  These relations are iterated over in ascending order with respect to
  the ordering for the family the elements are contained in.
  
  [1X4.3-6 Enumerator[0m
  
  [2X> Enumerator( [0X[3XGB[0X[2X ) ________________________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis.
  
  [6XReturns:[0X  an  enumerat  that enumerates the relations making up the Groebner
            basis.
  
  These  relations should be enumerated in ascending order with respect to the
  ordering for the family the elements are contained in.
  
  [1X4.3-7 Nontips[0m
  
  [2X> Nontips( [0X[3XGB[0X[2X ) ___________________________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis.
  
  [6XReturns:[0X  a list of nontip elements for [3XGB[0m.
  
  In order to compute the nontip elements, the Groebner basis must be complete
  and  tip reduced, and there must be a finite number of nontips. If there are
  an infinite number of nontips, the operation returns `fail'.

  [2X> IsPrefixOfTipInTipIdeal( [0X[3XGB, R[0X[2X ) ________________________________[0Xoperation
  
  Arguments: [3XGB[0m -- a Groebner basis, [3XR[0m -- a relation.
